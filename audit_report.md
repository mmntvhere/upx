# Отчет по техническому аудиту проекта UPX

**Дата:** 11 марта 2026 г.
**Роль:** Senior Software Architect / Code Auditor
**Объект:** Fullstack-приложение (Laravel + React/Vite)

---

## 1️⃣ Критические проблемы (Critical)

### 🚨 Безопасность: Утечка секретов и ключей
- **Проблема:** Файл `.env` содержит реальные пароли (`DB_PASSWORD`) и ключ доступа к **DeepL API**. Если этот файл попал в Git — это критическая уязвимость.
- **Риск:** Кража данных, финансовые потери из-за несанкционированного использования DeepL API.
- **Рекомендация:** Немедленно сбросить все ключи и пароли. Добавить `.env` в `.gitignore`. Использовать переменные окружения на хостинге.

### 💉 Безопасность: Потенциальная SQL-инъекция
- **Локация:** `CategoryController@index`, `PageController@index`.
- **Проблема:** Параметры `$request->get('sort')` и `$request->get('direction')` попадают напрямую в метод `orderBy()`.
- **Риск:** Злоумышленник может внедрить SQL-код через URL в админ-панели.
- **Рекомендация:** Добавить строгую валидацию (whitelist) для имен колонок: `in:id,name,slug,etc`.

### 🐢 Производительность: N+1 в сортировке
- **Локация:** `SiteSortController@sort`.
- **Проблема:** Внутри цикла `foreach` идет запрос к БД `Site::find($item['id'])` и последующий `save()`. При сортировке 50 сайтов это 100 запросов к БД.
- **Риск:** Зависание сервера при росте количества сайтов.
- **Рекомендация:** Использовать массовое обновление или загрузить все сайты одним запросом (`whereIn`) перед циклом.

---

## 2️⃣ Важные проблемы (Important)

### 🧩 Избыточное дублирование (Hardcode)
- **Проблема:** Список поддерживаемых языков (`en, ru, uk...`) жестко прописан (hardcoded) минимум в 4 местах: `AppRouter.jsx`, `SitemapController.php`, `SiteController.php`, `web.php`.
- **Риск:** При добавлении нового языка придется менять код во всем проекте. Высокий риск рассинхронизации.
- **Рекомендация:** Создать единый конфиг на бэкенде или таблицу в БД и отдавать его через API.

### 🔄 Лишние запросы на Frontend
- **Локация:** `StaticPage.jsx`.
- **Проблема:** При каждом переходе по слагам вызывается `Promise.all([fetchPages(), fetchPageBySlug(slug)])`. Список всех страниц (`fetchPages`) не меняется — его нужно кэшировать или брать из контекста.
- **Рекомендация:** Использовать React Query (TanStack Query) для автоматического кэширования API запросов.

---

## 3️⃣ Потенциальные проблемы (Minor)

### 📉 Масштабируемость API
- **Проблема:** Отсутствует Rate Limiting на публичных эндпоинтах (`/api/categories`, `/api/banners`).
- **Риск:** Парсинг (scraping) контента или DoS-атака положит сервер.
- **Рекомендация:** Настроить стандартный Laravel Middleware `throttle`.

### 🧪 Типизация
- **Проблема:** Весь фронтенд на чисто за JavaScript без JSDoc или TypeScript.
- **Риск:** Ошибки "Cannot read property of undefined" на проде при изменении структуры API.

---

## 4️⃣ Архитектурные рекомендации

1. **Единый сервис переводов:** Сейчас логика Spatie Translatable размазана по контроллерам. Стоит вынести сохранение переводов в базовый сервис или Trait.
2. **Маршрутизация:** В `AppRouter.jsx` огромный массив языков. Лучше использовать динамический параметр `:lang` в начале пути (`/:lang/*`), чтобы не дублировать маршруты вручную.
3. **Admin API:** Рекомендуется разделить API для фронта и логику админки (сейчас `SiteController` делает слишком много).

---

## 5️⃣ Улучшения производительности

- **Backend:** Включить Redis для кэширования ответов `/api/categories`. Данные меняются редко, а собираются тяжело (с подзапросами).
- **Frontend:** Оптимизировать `CategoryContext`. Сейчас он делает рефетч при любой смене языка, даже если данные уже есть в кэше.

---

## 6️⃣ Улучшения безопасности

- Удалить `APP_DEBUG=true` на продакшене (сейчас в `.env` стоит true).
- Настроить CSRF-защиту для всех POST-запросов (в `SiteSortController` проверка может отсутствовать, если Middleware не настроен корректно для API сегмента).

---

## 7️⃣ Общая оценка: 6/10

**Вердикт:** Проект рабочий и имеет хорошую базу (Laravel 12, Vite, Spatie Translatable), но выглядит как "быстрая разработка". Для запуска на большой трафик **критически важно** устранить SQL-инъекции в админке, убрать секреты из конфигов и оптимизировать количество запросов к БД.
